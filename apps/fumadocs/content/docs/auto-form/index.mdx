**auto-form-builder** génère des formulaires React complets à partir d'un
JSON Schema. Il utilise le **FormAdapter pattern** pour s'interfacer avec
n'importe quel form manager.

```tsx
import { AutoForm, tanstackAdapter, createShadcnRegistry } from "@code2-base-ui/auto-form-builder"

const schema = {
  type: "object",
  properties: {
    name: { type: "string", title: "Nom" },
    email: { type: "string", format: "email", title: "Email" },
    role: { type: "string", enum: ["admin", "user"], title: "Rôle" },
  },
  required: ["name", "email"],
}

<AutoForm
  adapter={tanstackAdapter}
  registry={createShadcnRegistry()}
  schema={schema}
  onSubmit={(data) => console.log(data)}
/>
```

## Architecture

Le package s'articule autour de 4 concepts :

| Concept | Rôle |
|---|---|
| **FormAdapter** | Interface entre le form manager et le rendu |
| **FieldRegistry** | Résout chaque champ en composant React |
| **FormLayout** | Stratégie de rendu (groupes, tableaux, unions) |
| **AutoForm** / **AutoFormBuilder** | Point d'entrée assembleur |

### FormAdapter

L'interface `FormAdapter` (2 composants) standardise la communication avec
n'importe quel form manager :

```ts
interface FormAdapter {
  name: string
  FormProvider: (props: FormProviderProps) => ReactNode
  Field: (props: FieldProps) => ReactNode
}
```

Trois adaptateurs sont fournis :
- `tanstackAdapter` — basé sur `@tanstack/react-form`
- `rhfAdapter` — basé sur `react-hook-form`
- `formischAdapter` — basé sur `@formisch/react`

### FieldRegistry

Enregistre les composants de champ par sélecteur (type / format / widget) :

```ts
const registry = createShadcnRegistry()
// Équivalent à :
const registry = new FieldRegistry()
registry.register({ type: "string", kind: "primitive" }, ShadcnTextField)
registry.register({ type: "string", kind: "enum" }, ShadcnEnumField)
registry.register({ type: "string", widget: "textarea" }, ShadcnTextareaField)
// ...
```

### FormLayout

Injecte la stratégie de rendu des conteneurs (groupes, tableaux, unions) :

```ts
const layout: FormLayout = { ...shadcnLayout, SubmitButton: MyButton }
```

### Sous-chemins d'import

```ts
import { tanstackAdapter } from "@code2-base-ui/auto-form-builder"
import { tanstackAdapter } from "@code2-base-ui/auto-form-builder/adapters/tanstack"
import { rhfAdapter }     from "@code2-base-ui/auto-form-builder/adapters/rhf"
import { formischAdapter } from "@code2-base-ui/auto-form-builder/adapters/formisch"
import { shadcnLayout }   from "@code2-base-ui/auto-form-builder/layout"
import { createShadcnRegistry } from "@code2-base-ui/auto-form-builder/fields"
import { mockAdapter }    from "@code2-base-ui/auto-form-builder/testing"
```

## Fonctionnalités

- Génération automatique des champs depuis JSON Schema
- 3 form managers supportés (TanStack Form, RHF, Formisch)
- Layout personnalisable (groupes, tableaux, unions)
- Composants shadcn/ui prêts à l'emploi
- Mode composé : contrôle manuel du rendu
- Validation intégrée via JSON Schema
- Champs conditionnels (unions)
- Wysiwyg `createAutoForm` pour usage simple
- Render-prop `AutoFormBuilder` pour usage avancé
