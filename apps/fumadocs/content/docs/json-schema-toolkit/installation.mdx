Deux méthodes d'installation sont disponibles.

## Via npm (package monorepo)

Dans un projet qui a déjà accès au workspace :

```bash
pnpm add @code2-base-ui/json-schema-toolkit
```

## Via GitHub Registry (shadcn)

Depuis n'importe quel projet, sans dépendre du monorepo :

```bash
npx shadcn@latest add WAROL52/code2-base-ui/json-schema-toolkit
```

Cette méthode copie les fichiers source directement dans votre projet
(sous `src/lib/json-schema-toolkit/`) et installe les dépendances npm
nécessaires (`@sinclair/typebox`, `@standard-schema/spec`).

### Items disponibles

```bash
npx shadcn@latest list WAROL52/code2-base-ui
```

| Item | Description |
|---|---|
| `json-schema-toolkit` | Package complet (10 fichiers, Standard Schema + utils + adapter) |
| `json-schema-toolkit-registry` | FieldRegistry React uniquement |
| `ui-button`, `ui-card`, ... | Composants shadcn/ui style base-lyra |

### Aperçu avant installation

```bash
npx shadcn@latest view WAROL52/code2-base-ui/json-schema-toolkit
npx shadcn@latest add WAROL52/code2-base-ui/json-schema-toolkit --dry-run
```

Le package fonctionne en environnement **Node.js** et **navigateur**.

## Dépendances

- `@sinclair/typebox` — Définition typée de schémas et validation runtime
- `@standard-schema/spec` — Types pour la conformité Standard Schema
- `react` (optionnel) — Requis uniquement pour `FieldRegistry`

## Configuration TypeScript

Le package utilise `verbatimModuleSyntax`. Assurez-vous que votre
`tsconfig.json` inclut :

```json
{
	"compilerOptions": {
		"verbatimModuleSyntax": true,
		"moduleResolution": "bundler",
		"module": "preserve"
	}
}
```

## Import

```ts
import { createStandardSchema, stringSchema, flatfields } from "@code2-base-ui/json-schema-toolkit";

// Sous-chemins
import type { SchemaAdapter } from "@code2-base-ui/json-schema-toolkit/adapter";
import { FieldRegistry } from "@code2-base-ui/json-schema-toolkit/registry";
```
