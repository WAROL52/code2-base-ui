5. Forces de l'architecture actuelle

✅ À garder absolument
  1. Plugin architecture — 5 packages séparés vs monolithe old
  2. createAutoForm factory — DI-friendly, testable, extensible
  3. FormStateAdapter abstraction — découplage form library
  4. React Context — AutoFormContextProvider + useAutoFormContext
  5. LayoutStrategy — disposition extensible, remplace le rendu hardcodé old
  6. FieldProvider — context-based field injection (vs render-props TanStack)
  7. SchemaProvider abstraction — agnostic ; old était lié à JSON Schema
  8. z.toJSONSchema() — délégation Zod v4 propre (vs mapping manuel old)

6. Recommandations

6.1 Blocants

B1 — Connecter les valeurs et onChange (T3) 🔴 Haute
tanstack-adapter field-components Sans ça, les inputs sont décoratifs. Chaque composant doit lire value, onChange, onBlur, error depuis useFieldContext(). Old faisait ça via form.Field render-prop. Current doit utiliser le contexte.
B2 — Validation par champ (T2) 🔴 Haute
tanstack-adapter Valider chaque champ sur onChange via provider.getFieldMeta(). Old utilisait validators.onChange.

6.2 Prioritaire

P1 — Standardized FieldComponentProps 🟡 Moyenne
auto-form-render-shadcn Old a une interface FieldComponentProps claire (field, label, value, onChange, error, placeholder, id, className, disabled). Current passe {...fieldMeta} sans contrat. Créer une interface partagée et typer les composants.
P2 — Options enum → Select/Radio (T5 suite) 🟡 Moyenne
field-components Les composants SelectField et RadioGroupField attendent options: { value, label }[] mais reçoivent enum: string[]. Convertir dans le composant ou normaliser dans flatfields. Nécessite T3 pour être fonctionnel.
P3 — Bouton submit par défaut (T6) 🟡 Moyenne
factory Ajouter un submit button quand children n'est pas fourni. Old l'avait par défaut.
P4 — Gestion récursive des objets (T7) 🟡 Moyenne
factory field-group AutoField doit détecter type: "object" et rendre FieldGroup avec sous-champs. Old gérait ça récursivement.

6.3 Polish

P5 — Adaptateur TanStack réel (T8) 🔵 Basse
Remplacer le stub in-memory par le vrai @tanstack/react-form. Dépend de T2+T3.
P6 — i18n (T9) 🔵 Basse
Copier old/core/i18n.ts dans le ZodProvider ou json-schema-toolkit.
P7 — x-ui-* via Zod meta (T10) 🔵 Basse
Lire z.string().meta({ widget: "textarea" }) via z.toJSONSchema() qui remplit déjà title et description.
P8 — Type guards (T13) 🔵 Basse
Exporter isFieldPrimitive, isFieldObject, isFieldArray, isFieldEnum depuis json-schema-toolkit.

8. Découvertes du review T1

Leçons apprises
  1. Les nouveaux fichiers doivent utiliser des tabs (convention Biome) — les fichiers humanize-path.ts et son test ont dû être réécrits.
  2. z.toJSONSchema() est un drop-in propre qui remplace 55 lignes de mapping manuel.
  3. Label obligatoire dans FieldMeta casse tous les tests qui construisent des FieldMeta — 8 fichiers impactés. C'est un bon signe de type safety.
  4. 96 tests couvrent 5 packages — le filet de sécurité est solide.
  5. Le pre-commit hook Husky échoue sur certaines configs — utiliser HUSKY=0 pour contourner.
